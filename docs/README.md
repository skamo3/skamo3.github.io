# docs 폴더 구조

카테고리별로 분리해 AGENTS.md(및 이를 읽는 에이전트)가 필요한 문서만 정확히 찾아갈 수 있도록 정리함.

- **`content/`** — 사이트에 실릴 콘텐츠의 소스 오브 트루스 (이력서/포트폴리오/Notion 요약 등)
- **`architecture/`** — 기술 스택/구조 결정 기록 (Jekyll 선택 이유, 대안 검토, 향후 계획)
- **`design/`** — 디자인 레퍼런스/근거 자료
- **`dev/`** — 로컬 개발 환경, 실행/빌드 방법
- **`harness/`** — 이 프로젝트에 적용한 하네스 엔지니어링 자체에 대한 메타 문서
- **`translation/`** — 한국어/영어 번역 파이프라인 등 상호작용 방식에 대한 조사/제안 (적용 여부 미정인 것 포함)

새 문서를 추가할 때는 위 카테고리 중 하나에 넣고, 어디에도 안 맞으면 새 카테고리를 만들기 전에 이 README를 먼저 갱신한다.
