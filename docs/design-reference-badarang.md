# 디자인 레퍼런스: badarang.netlify.app

Playwright(headless Chromium)로 실제 렌더링해 스크롤 순서/텍스트/색상을 확인함 (2026.07 기준, JS 렌더링 SPA라 일반 fetch로는 확인 불가했음).

## 스크롤 구조 (실측)
1. **고정 상단 네비게이션**: 로고+사이트명(좌) / 소개·프로젝트·문의하기 링크(우)
2. **Hero**: 2줄 타이틀("Badarang," 흰색 + "Indie Game Dev" 블루 강조) + 태그라인 2줄 + 이모지 포인트. 우측에 커스텀 3D 일러스트(책상+캐릭터, 데스크 셋업)
3. **About(소개)**: "안녕하세요!" 인삿말 + 서사형 자기소개 문단(성장 스토리) + 목표 문장. 아래 역할 뱃지 카드 2개(게임 클라이언트 프로그래머 / 테크니컬 아티스트), 아이콘+텍스트, 네이비 카드+블루 테두리
4. **Timeline(나의 게임 연대기)**: 섹션 소제목 + 큰 타이틀. 세로선 + 원형 마커로 5단계 연결(꿈의 시작→지식 확장→꿈 구체화→재미 탐구→꿈의 실현), 각 단계 카드에 소제목/기간/불릿 3개. 카드가 타임라인 좌우로 번갈아 배치
5. **기술 스택 아이콘 행**: 타임라인 바로 아래, HTML5/CSS3/JS/Python/C#/C++/(게임엔진 로고) 아이콘을 가로로 나열 (별도 섹션 타이틀 없이 삽입)
6. **Projects(만든 게임들)**: 섹션 소개 문구(정렬 기준 명시: 제작일 내림차순) + 3열 카드 그리드. 카드 = 정사각 썸네일(실제 게임 스크린샷) + 우상단 플랫폼 아이콘 뱃지(Steam/Google Play 등) + 제목 + 1줄 설명 + 장르 해시태그(카드마다 다른 포인트 컬러 사용: 핑크/그린/퍼플/옐로 등)
7. **Contact(문의하기)**: 2단 레이아웃. 좌측 네이비 폼 카드(이름/이메일/내용 입력 + 전송 버튼), 우측 별밤 배경 위에 3D 캐릭터+지구본 일러스트

## 폰트/타이포그래피 (실측, computed style)
- 기본 폰트: **Poppins** (구글 폰트), 한글은 시스템 폴백 (Noto Sans KR류로 추정)
- 헤딩(h1) font-weight: **900** (매우 굵게) — 임팩트 있는 타이틀 스타일
- 전체적으로 라운드하고 통통한 느낌의 굵은 산세리프

## 색상 (스크린샷 기반 근사치 — 정확한 HEX는 실제 CSS 미공개로 근사값)
- 배경: 거의 검정에 가까운 다크 (`#0B0B0D` ~ `#0E0F12` 근사)
- 카드/패널: 네이비 톤 (`#16213E` ~ `#1B2547` 근사), 옅은 블루 테두리
- 주 포인트색(타이틀 강조, 네비 링크, 타임라인 라인): 밝은 블루 (`#4C8DFF` ~ `#5B91F5` 근사)
- 프로젝트 태그: 카드마다 다른 색(핑크/그린/퍼플/옐로/시안 등)을 사용해 장르를 시각적으로 구분 — 단일 포인트색이 아니라 **카테고리별 멀티 컬러 태그** 전략
- 텍스트: 흰색(제목), 옅은 블루/그레이(보조 설명)

## 이 프로젝트에 참고할 점
- **구조**: Hero → About(자기소개+역할뱃지) → Timeline(세로선+단계카드) → 기술스택 아이콘 → Projects(카드그리드+장르태그) → Contact(폼+일러스트) 순서는 그대로 채택할 만함
- **차별화 포인트**: badarang은 인디 게임 개발자(1인 게임 출시 다수)라 게임 스크린샷 그리드가 강점. 김주찬은 엔진/그래픽스/툴 개발자에 가까우므로 프로젝트 카드에 "게임 플레이 스크린샷"보다 "엔진 데모 영상/렌더링 결과/성능 그래프" 등을 썸네일로 활용하는 편이 강점을 더 잘 보여줌
- **태그 전략**: 장르 해시태그 대신 기술 태그(#UE5 #DirectX11 #RayTracing #오디오 등)를 카테고리별 색상으로 구분하는 방식으로 응용 가능
- **폰트**: Poppins(영문/숫자) + Noto Sans KR(한글) 조합, 헤딩 900 굵기 스타일은 그대로 차용해도 좋음 (게임/테크 업계에서 널리 쓰이는 조합)
- **Contact 섹션**: 실제 폼(이름/이메일/내용/전송)을 두는 방식이 인상적 — 다만 GitHub Pages는 정적 호스팅이라 폼 제출 처리에 Formspree/EmailJS 같은 외부 서비스 연동이 필요함 (스캐폴딩 단계에서 결정)
