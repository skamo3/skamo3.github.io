# 프로젝트 소개

42 Seoul 커리큘럼의 웹 프로그래밍 팀 프로젝트로, Ping Pong 게임 매칭과 채팅 시스템, 유저 정보 관리(닉네임·프로필사진·플레이 히스토리 등)를 갖춘 웹 애플리케이션. NestJS + PostgreSQL 백엔드와 React + TypeScript + MUI 프론트엔드로 구현했고, Docker/Docker Compose로 개발·배포 환경을 구성.

기획 단계에서 페이지별 와이어프레임을 먼저 그려 팀원들과 화면 흐름을 맞춘 뒤 구현에 들어감.

# 실시간 Ping-Pong 게임

실시간 대전 로직 자체는 팀원이 메인으로 구현했고, 로직 설계와 소켓 통신 부분을 함께 보조.

- 게임 로직은 서버가 권위를 가지고 동작 — 클라이언트는 각자 패들 이동 입력을 소켓으로 전송할 뿐, 실제 위치·충돌 판정 등 게임 상태 갱신은 서버에서 처리
- 초기에는 로직을 프론트엔드에 두고 개발했지만, 클라이언트 조작에 의한 결과 조작 가능성(무결성)과 두 클라이언트 간 상태 불일치(안정성) 문제 때문에 서버 권위 구조로 로직을 이전

# 채팅방 관리

4인 중 채팅 시스템을 전담. 프론트/백엔드 모두 담당.

- 입장 가능한 채팅방 리스트 관리 — Public/Private 권한에 따라 Visible 여부 관리
- 채팅방 입장 시 이전 대화내용 Load 및 SocketIO를 이용한 채팅 소켓 접속
- 채팅방 내 유저 별 권한에 따른 접근제어 설정 — Admin/Manager/User에 따라 각자 할 수 있는 기능의 Visible 상태를 다르게 관리

권한별 UI를 결정하는 React state 관리에서 초기에 많이 헤맴. State 관리에 익숙하지 않아 데이터가 꼬이는 문제를 여러 번 겪었고, JSX return 문 안에서 다중 삼항 연산자로 조건 분기를 처리하려다 로직이 꼬여버린 적도 있었다. 잘못된 접근임을 깨닫고 여러 차례 다시 풀어냈지만, 이미 얽혀버린 로직을 정리하는 데 애를 먹음.

채팅방 입장 시에는 최근 대화 내용 일부만 우선 불러오고, 새 메시지가 오면 자동으로 스크롤이 맨 아래로 포커싱되도록 구현. 사용자가 스크롤을 위로 올려 채팅 목록 최상단에 닿으면 그때 서버에 이전 대화를 추가로 요청하는, 페이지네이션과 유사한 방식으로 처리(소켓 재연결 시 메시지 유실·중복은 프로젝트 규모상 크게 문제되지 않아 별도로 처리하지 않음).

Private 방 접근 제어에서는 조건 분기가 여러 컴포넌트에 흩어져 있다 보니, 특정 행동 이후 Private 방에 접근 가능해지는 버그를 겪은 적이 있다. 원인은 다른 컴포넌트가 같은 조건을 건드리면서 생긴 것으로, 이 경험으로 접근 제어 조건은 한 곳에서 명확히 분리해 관리하는 게 중요하다는 걸 체감.

# 메인 페이지와 Admin 페이지

메인 페이지에서는 게임 매칭과 팔로우 중인 유저의 접속 상태를 확인 가능. 온라인 상태 표시를 처음 구현했을 때는 새로고침을 해야만 상태가 갱신되는 문제가 있었고, 이를 고치는 과정에서 소켓 연결 범위를 어디에 둘지 고민하게 됨. 전역에서 미리 소켓을 연결해두는 방식도 고려했지만, 팔로우 유저 목록을 보지 않는 상황에서도 계속 연결이 유지되는 게 비효율적이라 판단해, 팔로우 유저 목록 컴포넌트가 렌더링될 때만 소켓에 접속하도록 설계 — 컴포넌트의 책임 범위와도 맞는 방식이라고 생각함.

Admin 페이지는 사이트 전체에 대한 관리 권한을 부여.

# 회고

Jira로 각자 태스크를 관리하며 작업 중 생기는 이슈를 계속 등록하고, 프론트/백엔드가 서로의 이슈에 태그를 걸어 확인하는 방식으로 협업. 개발과 QA를 동시에 진행했고, 3개월 내 완료라는 목표가 있어 막바지에는 기능 구현 속도를 최대한 높이고 QA에 더 집중해 실제로 잘 작동하는 기능을 만드는 데 집중. 팀원들과 합이 잘 맞아 일정 조율 실패 없이 마무리한 프로젝트.

![Jira 스프린트와 이슈 트래킹 화면](/assets/images/pongski/jira-sprint.png){: width="90%"}

React로 상태관리와 컴포넌트 단위 기능 분리를 직접 설계해보며 프로그램 설계 능력을 키울 수 있었고, 팀 프로젝트에서 이슈 관리와 일정 조율, 원활한 커뮤니케이션이 왜 중요한지 체감한 프로젝트.
