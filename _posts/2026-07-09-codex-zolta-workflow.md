Zolta는 OpenAI의 Codex CLI로 만들고 있는 UE5 프로젝트다. Claude Code로 포트폴리오 사이트를 만들 때와는 또 다른 방식으로 협업하게 된다. 5개월 넘게 같이 작업하면서 Codex를 어떻게 길들이고 다뤄왔는지, 그 고민 몇 가지를 정리해본다.

## 방향만 던져도 알아서 문서부터 찾는다

구조를 크게 바꾸고 싶을 때 나는 보통 이런 식으로 가보자는 한 문장만 던진다. 캐릭터 기획 변경이라고만 말했을 때도 Codex는 바로 코드를 고치지 않았다. 프로젝트 안의 기획서 파일부터 찾아 읽고, 캐릭터 시스템 챕터만 바꿀 건지 스킬과 UI까지 포함인지, 이번 목표가 초안 정리인지 실제 구현 가능한 수치 기획인지부터 되물었다. 무작정 손대지 않고 범위를 먼저 확인하는 습관이 붙어있어서 큰 방향 전환에서도 엉뚱한 곳까지 건드리는 일이 적었다.

## 문서를 계속 다시 잡았다

이 프로젝트는 시작할 때부터 하네스 엔지니어링을 적용했다. AI가 작업할 때 참고할 규칙과 기록을 문서로 미리 갖춰두는 방식이다. AGENTS.md를 기본 규칙 파일로 두고 docs 아래에 기획 문서와 프로그래밍 관련 문서를 나눠뒀다. 코딩 스타일, 작업 패턴, 검증 규칙, 실패 기록, 개발 일지 이렇게 나눴다.

그런데 몇 달 지나니 이 문서들이 계속 불어났다. 그래서 6월에 한 번 크게 정리했다. 이제부터 할 일은 문서를 줄이고 통합해서 토큰 사용량을 줄이는 것이라고 방향을 잡았다. 안 쓰는 스펙 문서는 지우고, 개발 일지는 작업이 끝났을 때만 TodoList와 짧은 누적 기록만 남기는 형태로 단순화했다. 상세한 추적은 git 커밋 메시지에 맡기고 일지 자체는 요약만 적는 쪽으로 역할을 나눴다. AGENTS.md에도 짧게 쓴다, 같은 정보를 두 번 적지 않는다 같은 토큰 절감 원칙을 아예 명문화해뒀다.

## 시킨 것 이상 하지 않기

한번은 월드 아이템 상호작용 구조를 정리하다가 Codex가 필요하면 이름과 색상 표시용 함수도 추가해두겠다고 한 적이 있다. 그때 바로 제동을 걸었다. 시킨 것 이상으로 미리 준비해두는 것 자체가 지시 이외의 행동이라고 짚었다. 그 뒤로 AGENTS.md에 지시받은 범위 외 코드는 만들지 않는다, 불필요한 if 검사는 피하고 꼭 필요한 최소 조건만 둔다는 원칙을 규칙으로 박아뒀다.

돌아보면 이 프로젝트에서 구조를 여러 번 다시 만든 것도 결국 일단 확장성 있게 만들어두자는 유혹에 넘어갔다가 되돌아온 경우가 많았다. InteractionActor에서 ItemActor를 거쳐 WeaponActor로 이어지는 3단 상속 구조를 만들었다가 나중에 중간 계층을 다시 없앤 것도 같은 맥락이다. AI에게도 나 스스로에게도 지금 필요한 만큼만이 계속 되새겨야 하는 원칙이라는 걸 느꼈다.

## Codex는 직접 플레이해볼 수 없다

Codex는 코드를 고칠 순 있지만 직접 언리얼 에디터를 켜고 캐릭터를 움직여볼 수는 없다. 그래서 내가 원하는 대로 동작하는지를 어떻게 검증할지가 계속 고민이었다. 결국 기능을 구현할 때마다 Codex가 DoD를 같이 제시하는 흐름으로 정리했다. 이 조건을 만족하면 완료로 본다는 체크리스트다. 나는 그 DoD를 기준으로 직접 에디터에서 플레이 테스트를 해본다. Codex가 여기까지 구현했고 이렇게 테스트해보면 된다고 알려주면 내가 실제로 눌러보고 결과를 다시 알려주는 식으로 주고받았다.

## 새 도구도 같이 실험해봤다

작업 중간중간 Unreal MCP도 붙여봤다. Codex 같은 AI가 언리얼 에디터의 에셋과 블루프린트 구조에 직접 접근할 수 있게 해주는 프로토콜이다. uasset이나 블루프린트를 분석해야 할 때는 MCP로 먼저 구조를 확인하도록 AGENTS.md에 규칙을 추가했다. 관련 스킬도 상황별로 나눠서 쓰도록 정리했다. 에디터 작업은 ue5-editor-control, 개념 정리는 unreal-engine, C++ 리팩터링은 unreal-engine-cpp-pro 이런 식이다.

## 이 글을 쓰게 된 계기

사실 이 정리도 어느 날 지금까지 작업한 걸 블로그에 파트별로 정리해서 내 고민과 고찰을 남겨볼까 싶어서 Codex에게 지난 세션들을 다시 훑어달라고 부탁한 데서 시작됐다. 실제로 시켜보니 그날그날의 대화 속에 묻혀 있던 결정들이 생각보다 잘 남아있었다. 왜 이렇게 바꿨는지가 대화에 그대로 남아있었다. 문서에는 최종 결론만 짧게 적혀 있지만 실제 고민의 흔적은 대화 기록 쪽에 훨씬 많이 남아있다는 것도 이번에 새삼 느꼈다.
